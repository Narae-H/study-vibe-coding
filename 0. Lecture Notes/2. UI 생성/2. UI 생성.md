# `src/components`

## 컴포넌트 및 주소(URI) 설계 설명

### 1. `Components` 폴더의 역할
- `src/components` 아래에 폴더를 만든다고 해서, **그 컴포넌트가 자동으로 브라우저에 표시되는 것은 아님**.
- 브라우저에서 특정 주소(예: `/diaries`)를 입력했을 때 해당 컴포넌트를 보여주려면,**라우팅 설정(주소 설계)** 를 해야 함.
- 즉, **주소(URI)를 설계하고 해당 경로에 컴포넌트를 매핑**해야 화면에 표시됨.

### 2. 주소 설계

| 주소            | 설명                  |
| ------------- | ------------------- |
| `/diaries`    | 다이어리 목록 페이지         |
| `/diaries/17` | 다이어리 상세 페이지 (id=17) |

### 3. App Router에서 페이지 연결 방식
**Next.js**의 **App Router 구조**를 사용할 경우, `/app` 폴더 안에서 라우팅 구조를 직접 폴더로 설계함.

#### 예시
- 📄 목록 페이지
  ```bash
  /app/diaries/page.tsx
  ```
  - `/diaries` 주소로 접근했을 때 렌더링됨
  - 내부에서 `components/diaries` 컴포넌트를 import하여 사용

- 📄 상세 페이지 (Dynamic Routing)
  ```bash
  /app/diaries/[id]/page.tsx
  ```
  - `/diaries/17` 처럼 id가 동적인 주소를 처리
  - 내부에서 `components/diaries-detail` 컴포넌트를 import하여 사용

- 📄 신규 작성 (Modal)
  - `/diaries-new`와 같은 모달은 **경로를 갖는 경우도 있고, 없는 경우도 있음**.
  - 이 프로젝트에서는 **별도의 페이지 경로 없이** 모달로만 표시하도록 구현함.

### 4. 개발 시 주의사항
- `app` 폴더 아래에 실제로 존재하지 않는 페이지 파일(`page.tsx`)을 미리 만들어두면 빌드 오류가 발생할 수 있음.
- 빌드 시 빈 파일이 있으면 에러가 날 수 있으므로, 아래 순서로 개발을 진행하는 것이 안전함
  - **실제로 필요한 파일만 생성 → 빌드 확인 → 다음 파일 추가**

### 5. 폴더 구조 예시

```bash
📂 프로젝트 루트
├── 📂 app
│   ├── 📂 diaries
│   │   └── 📄 page.tsx              # /diaries
│   ├── 📂 diaries
│   │   └── 📂 [id]
│   │       └── 📄 page.tsx          # /diaries/[id]
├── 📂 src
│   ├── 📂 components
│   │   ├── 📂 diaries
│   │   ├── 📂 diaries-detail
│   │   ├── 📂 diaries-new
│   │
```
