# 반응형 디자인

## 1. 개요

- **반응형 디자인**: 화면 크기(데스크톱, 태블릿, 모바일 등)에 따라 레이아웃이 자연스럽게 변하도록 만드는 기법
- 이번 강의에서는 **비율 기반 레이아웃**으로 먼저 전체 구조를 잡고, 이후 **브레이크포인트 기반 레이아웃**으로 리팩터링하는 흐름으로 진행

<br/>

## 2. 반응형 디자인의 두 가지 접근

### 2-1. 비율 기반 레이아웃 (%)

- **개념**
  - `width: 100%`, `flex: 1` 같은 비율(%)과 유연한 단위를 중심으로 레이아웃을 구성
  - 고정 px 값 대신 **비율로 영역을 나누는 방식**
- **특징**
  - 동일한 코드로도 어느 정도 화면 크기에 대응 가능
  - 기존 레이아웃을 크게 바꾸지 않고, **기존 코드 수정** 위주로 적용하기 좋음

### 2-2. 브레이크포인트 기반 레이아웃

- **개념**
  - 화면 너비(예: `768px`, `1024px`)를 기준으로 **구간별 스타일을 분리**하는 방식
  - CSS `@media` 쿼리나 Tailwind의 `sm:`, `md:`, `lg:` 같은 유틸 클래스를 주로 사용
- **특징**
  - 특정 기기(모바일, 태블릿, 데스크톱)에 최적화된 레이아웃을 설계할 수 있음
  - 새로운 구간별 스타일이 들어가므로, **신규 코드 추가** 개념에 가깝다.
- **진행 전략**
  - 먼저 **비율 기반 레이아웃으로 전체 구조를 안정화**한 뒤
  - 이후에 **브레이크포인트를 추가**하면서 세부 레이아웃을 조정

<br/>

## 3. 병렬 처리 진행 순서 (레이아웃 작업)

이전 **병렬 처리** 노트에서처럼, 여기서도 페이지/영역을 나눠서 병렬로 작업하고,  
마지막에 한 번에 스타일 정리 및 테스트를 진행하는 흐름으로 생각할 수 있다.

### 3-1. 페이지·영역별 작업 매트릭스

아래 표는 **비율 기반 → 브레이크포인트 기반** 순서로 레이아웃을 구현할 때의 병렬 처리 관점을 정리한 것:

|전체 레이아웃 (비율) | 일기페이지: 검색영역(비율) | - | 일기상세페이지: 전체영역(비율) | 사진페이지: 전체영역(비율) |
| --- | --- | --- | --- | --- | 
|  | 일기페이지: 메인영역(비율) |   |   |   |
|  | 일기페이지: 검색영역(브레이크포인트) | | 일기상세페이지: 전체영역(브레이크 포인트) | 사진페이지: 전체영역(브레이크 포인트) |
|  | 일기페이지: 메인영역(브레이크포인트) |   |   | 

- 1단계(비율 기반)는 **공통 레이아웃 구조를 빠르게 맞추는 단계**
- 2단계(브레이크포인트 기반)는 **디바이스별 경험을 세밀하게 다듬는 단계**
