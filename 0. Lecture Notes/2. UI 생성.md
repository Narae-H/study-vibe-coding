# UI 생성 순서

### 1. `Wireframe` 구조(큰 그림)를 먼저 잡기
- Wireframe: Figma 파일 대로 구현하기 전에 뼈대를 미리 잡아서 실제 디자인이 어디다가 구현되면 될지 알려주는 역할

### 2. 생성된 Wireframe 위에 생성하고 싶은 컴포넌트의 `Figma 파일`을 가져와서 입히기
- 핵심 포인트: 
  - 이미 만들어둔 **공통 컴포넌트**를 사용 ex. 버튼(Button), 카드(Card)
  - 새로운 컴포넌트를 생성할 때는, 기존 공통 컴포넌트의 **디자인 원칙과 제약 조건을 고려**하여 프롬프트를 작성
  - 예시:

    | 속성      | 설명                             |
    | ------- | ------------------------------ |
    | **높이**  | 지정된 값 중 하나 선택 (`s`, `m`, `l`)  |
    | **너비**  | 동적으로 변경 가능                     |
    | **스타일** | `primary`, `secondary` 중 선택 가능 |
  - ⚠️ 버튼 생성 시에는 위 원칙 내에서만 커스터마이징을 허용. But, 높이와 스타일은 고정된 값 중에서만 선택

  - 검토 시에도, **공통 컴포넌트 검토 및 커서 룰 검토**라고 꼭, 공통 컴포넌트 넣어서 요청하기
  - 몇 번 추가 시도 후에도 결과가 만족스럽지 않으면, 반복되는 구조나 컴포넌트 단위(ex.카드 컴포넌트, 리스트 컴포넌트 등)등을 더 작은 단위로 쪼개서 생성

# 컴포넌트 UI 생성 - `src/components`
- 페이지를 구성하는 **단위 컴포넌트들**들을 지칭

  <details>

  <summary>컴포넌트 - 자세히</summary>

  ### 1. `src/components` 폴더의 역할
  - 페이지를 구성하는 **단위 컴포넌트들**을 모아두는 폴더 ex. 다이어리 리스트, 상세 카드, 네비게이션 바 등
  - **특정 페이지나 기능 내에서만 사용되는 부품 단위의 컴포넌트**를 포함
  - 이 폴더에 컴포넌트를 만든다고 해서 자동으로 화면에 표시되지는 않으며, 페이지(src/app 또는 src/pages)에서 임포트해 사용
  - 브라우저에서 특정 주소(예: `/diaries`)를 입력했을 때 해당 컴포넌트를 보여주려면,**라우팅 설정(주소 설계)** 를 하고, **해당 경로에 컴포넌트를 매핑**해야 화면에 표시됨.

  ### 2. 주소 설계

  | 주소            | 설명                  |
  | ------------- | ------------------- |
  | `/diaries`    | 다이어리 목록 페이지         |
  | `/diaries/17` | 다이어리 상세 페이지 (id=17) |

  ### 3. App Router에서 페이지 연결 방식
  **Next.js**의 **App Router 구조**를 사용할 경우, `/app` 폴더 안에서 라우팅 구조를 직접 폴더로 설계함.

  #### 예시
  - 📄 목록 페이지
    ```bash
    /app/diaries/page.tsx
    ```
    - `/diaries` 주소로 접근했을 때 렌더링됨
    - 내부에서 `components/diaries` 컴포넌트를 import하여 사용

  - 📄 상세 페이지 (Dynamic Routing)
    ```bash
    /app/diaries/[id]/page.tsx
    ```
    - `/diaries/17` 처럼 id가 동적인 주소를 처리
    - 내부에서 `components/diaries-detail` 컴포넌트를 import하여 사용

  - 📄 신규 작성 (Modal)
    - `/diaries-new`와 같은 모달은 **경로를 갖는 경우도 있고, 없는 경우도 있음**.
    - 이 프로젝트에서는 **별도의 페이지 경로 없이** 모달로만 표시하도록 구현함.

  ### 4. 개발 시 주의사항
  - `app` 폴더 아래에 실제로 존재하지 않는 페이지 파일(`page.tsx`)을 미리 만들어두면 빌드 오류가 발생할 수 있음.
  - 빌드 시 빈 파일이 있으면 에러가 날 수 있으므로, **실제로 필요한 파일만 생성 → 빌드 확인 → 다음 파일 추가**로 진행

  ### 5. 폴더 구조 예시

  ```bash
  📂 프로젝트 루트
  ├── 📂 app
  │   ├── 📂 diaries
  │   │   └── 📄 page.tsx              # /diaries
  │   ├── 📂 diaries
  │   │   └── 📂 [id]
  │   │       └── 📄 page.tsx          # /diaries/[id]
  ├── 📂 src
  │   ├── 📂 components
  │   │   ├── 📂 diaries                
  │   │   │   ├── 📄 index.tsx         # 여기서 컴포넌트 export & app/diaries에서 import
  │   │   │   └── 📄 styles.module.css 
  │   │   ├── 📂 diaries-detail
  │   │   │   ├── 📄 index.tsx         # 여기서 컴포넌트 export & app/diaries/[id]에서 import
  │   │   │   └── 📄 styles.module.css 
  │   │
  ```
